Wallstreetbets

LIVE DEMO GitHub

Wallstreetbets is a stock trading platform that offers real-time data for companies in the NASDAQ market and allows users to mimic buying, selling, and keeping track of stocks. Theme is based off the subreddit /r/wallstreetbets.

Features

Searchbar

The implementation of the searchbar was a great learning experience for managing redux slices of state using JBuilder, as well as using Ruby on Rails' active record to query into the database. I utilized an onchange action on the text input to simultaneously update the local state of the component, ultimately to send out the appropriate string query with the dispatched action.

Watchlist

The concept of building a watchlist was pretty simple. I structured the database schema so that a user had a watchlist and a watchlist had many watchlist items. However, it was challenging to constantly update the assets to reflect real-time market prices. I was able to implement this feature by using the built-in Redux Life Cycle methods. Upon the dashboard component mounting, I utilized a setInterval function that sent out an ajax request to the third party API every 25 seconds. When the user navigated away from that page, componentDidUnmount was triggered and the setInterval function was cleared.

Asset Research

The graph representing the asset's price history was made using a library called recharts which is built on D3.js. It was tricky formatting the data to correctly fit into the graph's data input. This was all achieved using JBuilder in the Ruby on Rails views of the MVC framework. Another tricky part of implementing the graphs was creating a custom tooltip position outside of the graph in addition to the default tooltip that followed the user's cursor when hovering over the graph. For this, I created a new component and a new slice of state that was updated with the user's mouse movement over the range of prices.

Particles

LIVE DEMO GitHub

Particles is a molecular inspired animation that allows the user to interact with particles on a canvas and manipulate their relationships to other particles based on proximity to each other and to the user's mouse. Users can select multiple forms of interactions, each based on a unique mathematical calculation.

    Technologies

  • Canvas
  • Javascript
  • CSS3
  • HTML5

Features

Repulsion

Implemented repulsion by taking into account of the angle of each particle relative to the mouse and the particle's quadrant. Utilized Arctan2 as opposed to Arctan to return the correct sign for y values when the particle was at an angle between 0 and π, and between 0 and -π. This following solution was implemented to ensure that the change in x and y of each particle had the correct signs to push the particle to the correct final position, outside the effective distance.

Attraction

Implemented attraction by making the dx and dy velocities of the particle dependent on the distance between the mouse and the particle. As the mouse and the particle become closer, the particle accelerates toward the mouse until the distance between the two become 0, resulting in final dx and dy particle velocities of 0. The particles ultimately become immobile at the site of the mouse until the user starts to move the mouse again.
